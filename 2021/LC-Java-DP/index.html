DP 思路


分治 （Divide and Conquer）

将原问题分解为若干个规模较小但类似于原问题的子问题（Divide），「递归」的求解这些子问题（Conquer），然后再合并这些子问题的解来建立原问题的解。

  • 用动态规划能解决的问题， 分治策略肯定能解决，只是运行时间长了。
  • 分治策略一般用来解决子问题相互对立的问题，称为标准分治
  • 动态规划用来解决子问题重叠的问题

DP & 分治:

  • 动态规划法试图只解决每个子问题一次
  • 一旦某个给定子问题的解已经算出，则将其记忆化存储，以便下次需要同一个子问题解之时直接查表。

思路

https://cloud.tencent.com/developer/article/1650234

​

1. 自下而上:

若采用递归这样自顶向下的求解方式，子问题之间可能存在大量的重叠，意味着大量地重复计算，这样时间复杂度很可能呈指数级上升，所以自下而上的求解方式可以消除重叠子问题。

动态规划一般都脱离了递归，而是由循环迭代完成计算。


2. 步骤：

1 ) 确定 Edge cases，另外一定要考虑：数组不要越界！ e.g. 定义了 MIN_VALUE, MAX_VALUE

​

2 ) 开数组，初始化 （初始条件：用转移方程算不出来的，并且需要定义的）

所有 DP 题目中，必须要给出至少一个初始条件！ e.g. t[0] = 0;


3 ) 最后一步 （t[i], t[i-1]）& 状态转移方程


3. 三类题型：

  • Min / Max
  • 有多少种方法 (++)
  • 是否存在 True / False (用 OR, AND)

4. 如何确定运算顺序：

怎么知道是从小到大，还是从大到小：

标准：当等式左边用 t[i] 的时候，右边用到的状态都已经算过了 （ t[i-1], t[i-2] ）


5. 开几维数组：

子问题里（ t[i-1] ），有一个变量，就一维数组。有2个变量，就二维数组

Beta Testing

题目可以抽象为将n个数字分为d段，每段中最大的数字代表本段，求所有段最大值之和的最小值。

Cut the array into d contiguous subarrays, return min sum of the max of each subarray.


把day天完成的任务，拆成 day-1天 加 1天 的子任务解决

t[day][j] = min(t[day][j], t[day - 1][k-1] + max(i, k))

Min-Max 组合，想要求得 Min difficulty，就要知道区间 （k+1, i）取的最大 difficulty


二维数组初始化直接用 for 循环更简单，不用 Arrays.fill

Time: O(d * len ^ 2)

Space: O(d * len)

Unique BSTs

看到题目中有 unique 这个词，就想到和 DP 有关！ 不要被题目中的 BST 所忽悠

推导：

  • 假设 n 个节点存在二叉排序树的个数是 G (n)，令 f(i) 为以 i 为根的二叉搜索树的个数，则

    ​ G(n)=f(1)+f(2)+f(3)+f(4)+…+f(n)

  • 当 k 为根节点时，其左子树节点为 k-1 个，右子树节点为 i-k，则 (i 相当于 n，现在以 k 为根结点)

    ​ f(i)=G(k−1)∗G(i−k)


卡塔兰数 ： (前5項的卡塔兰数為：1, 1, 2, 5, 14)

并且满足： 简化： (4n + 2) / (n + 2)

推导 (Link)： 这个是 f(n-1) 和 f(n) 的关系，要求 f(n+1), 只要在结果上用 (n+1) 就行

If Exists (T / F)


Jump Game

判断是否存在，想到 DP ，但是时间慢


Greedy， 推荐！ 为 O(n)

思想： 每次都选最大的，最后步骤一定最少

The basic idea is this: at each step, we keep track of the furthest reachable index. The nature of the problem (e.g. maximal jumps where you can hit a range of targets instead of singular jumps where you can only hit one target) is that for an index to be reachable, each of the previous indices have to be reachable.

Hence, it suffices that we iterate over each index, and If we ever encounter an index that is not reachable, we abort and return false. By the end, we will have iterated to the last index. If the loop finishes, then the last index is reachable.
